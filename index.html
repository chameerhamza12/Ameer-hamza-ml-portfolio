ML Algorithms Visual Framework


Title

Machine Learning Algorithm Visual Framework: A Comprehensive Classification of 11 Key AI/ML Algorithms by Learning Type, Application Domain, and Real-World Use Cases

Introduction

A professional portfolio piece for AIML-501 presenting a visually structured framework to help practitioners and employers understand the landscape of machine learning algorithms — how they are categorized, where they are applied, and how they relate across the AI field.

Description

The framework covers 11 key ML algorithms spanning supervised and unsupervised learning, organized across four major domains: Tabular Data, Computer Vision, NLP, and Generative AI. Each entry includes learning type, applicable domains, plain-English explanation, and real-world use cases. Also includes a relationship map, classification rationale, and comprehensive reflection.

Objective

To develop a clear, accurate, and visually compelling reference framework demonstrating mastery of key ML algorithm types and their practical applications — fulfilling course learning objectives while serving as a professional portfolio piece for potential employers.

Process

Structured multi-step process: (1) reviewing course materials; (2) selecting 11 representative algorithms; (3) researching each algorithm's mechanism and real-world applications; (4) designing classification system; (5) building visual framework in HTML/CSS; (6) writing classification rationale and reflection.

Tools and Technologies Used

HTML5, CSS3, JavaScript for visual design and animations; Google Fonts for typography; GitHub Pages for public portfolio hosting; course readings and AI/ML reference materials for content accuracy.

Value Proposition
Unique Value

Goes beyond a simple list — visually maps relationships between algorithms, explains classification reasoning, and positions every algorithm within both theoretical and practical real-world context.

Relevance

Navigating the algorithm landscape is a core professional competency. This artifact directly demonstrates that ability and is relevant to any future role involving ML model selection, AI strategy, or data science.

References
  • Goodfellow, I., Bengio, Y., & Courville, A. (2016). Deep learning. MIT Press.
  • Mitchell, T. M. (1997). Machine learning. McGraw-Hill.

Explanation of Classification Process
Why I Classified Algorithms This Way

Classification Rationale & Critical Thinking

Building this framework required deliberate decisions at three levels: learning type, application domain, and algorithm selection. Below is the explicit rationale for each major decision demonstrating the critical thinking behind the structure.

SupervisedLinear Regression

Classified as supervised because it requires a labeled continuous target variable. Assigned to tabular data because inputs are structured numerical features. Included as Algorithm #1 because it underpins the mathematics of neural networks.

SupervisedDecision Tree & Random Forest

Both supervised tabular. Random Forest included as a distinct entry because it fundamentally changes the bias-variance tradeoff through ensemble aggregation — overcoming the core weakness of a single tree.

SupervisedSVM

Placed across tabular and computer vision because SVMs were the dominant image classifier before CNNs and remain applicable in low-data CV scenarios — reflecting meaningful historical and practical cross-domain relevance.

SupervisedCNN & Transformer

CNN placed in CV and Generative AI because it forms the backbone of GAN generators. Transformer placed in NLP and Generative AI because BERT and GPT represent the same architecture applied to two distinct objectives.

SupervisedGAN & Diffusion Model

Both classified as supervised — correcting the common misconception that generative models are unsupervised. GANs use real/fake binary labels; Diffusion Models supervise the denoising process at each noise level.

UnsupervisedK-Means & PCA

Both unsupervised — neither requires class labels. PCA placed across tabular, CV, and NLP because dimensionality reduction is universally applicable — reducing noise in tabular features, image pixel spaces, and word embeddings.

Understanding Neural Networks

Neural networks are computational systems inspired by the human brain. They consist of interconnected layers of neurons that process information, learn patterns from data, and make predictions. This artifact explores the six core components of neural networks — from the basic building blocks to the algorithms that optimize their performance.

Core Components
6 Neural Network Components
🗂️
Component 01
Layers
Layers are the fundamental organizational units of a neural network. Every neural network consists of at least three types of layers: an input layer that receives raw data, one or more hidden layers that perform intermediate computations, and an output layer that produces the final prediction or classification.
How It Works
Each layer receives the output of the previous layer, applies a mathematical transformation, and passes the result forward. The input layer simply passes raw feature values into the network. Hidden layers extract increasingly abstract representations of the data — early hidden layers may detect simple patterns like edges in an image, while deeper layers detect complex concepts like faces. The output layer converts the final representation into the desired format — a class label, probability, or numerical value.
Types
Input Layer Hidden Layers Output Layer Convolutional Layer Recurrent Layer
⚡
Component 02
Neurons
Neurons (also called nodes or units) are the individual computational elements of a neural network, inspired by biological neurons in the human brain. Each neuron receives one or more input signals, computes a weighted sum of those inputs, adds a bias term, and passes the result through an activation function to produce an output signal.
How It Works
Mathematically, a neuron computes: output = activation(Σ(weight × input) + bias). The bias allows the neuron to shift its activation threshold, giving the network more flexibility to learn complex patterns. A single neuron on its own is equivalent to logistic regression. The power of neural networks comes from connecting thousands of neurons in parallel and in depth, enabling the representation of highly complex, non-linear functions.
Key Properties
Weighted sum Bias term Activation threshold Output signal
⚖️
Component 03
Weights
Weights are the learnable numerical parameters of a neural network — the values that the network adjusts during training to improve its predictions. Each connection between two neurons has an associated weight that determines how strongly the output of one neuron influences the input of the next. Weights encode all the knowledge the network has learned from training data.
How It Works
At the start of training, weights are initialized randomly. During each training iteration, the network makes a prediction, compares it to the true label using a loss function, and then uses backpropagation to calculate how much each weight contributed to the error. An optimization algorithm then updates each weight slightly in the direction that reduces the error. This process repeats thousands of times across the training dataset until the weights converge on values that produce accurate predictions.
Key Concepts
Random initialization Backpropagation Weight update Convergence
📈
Component 04
Activation Functions
Activation functions introduce non-linearity into the neural network, enabling it to learn and represent complex patterns that cannot be captured by any linear combination of inputs. Without activation functions, a neural network — regardless of how many layers it has — would be mathematically equivalent to a single linear transformation, severely limiting what it can learn.
How It Works
After a neuron computes its weighted sum, the activation function transforms that value before passing it to the next layer. ReLU (Rectified Linear Unit) outputs zero for negative inputs and the input itself for positive inputs — simple but highly effective. Sigmoid squashes values to between 0 and 1, useful for binary classification outputs. Softmax converts a vector of values into a probability distribution across multiple classes, ideal for multi-class output layers.
Common Functions
ReLU Sigmoid Softmax Tanh Leaky ReLU
🎯
Component 05
Loss Functions
A loss function (also called a cost function or objective function) measures how far the network's predictions are from the true target values. It produces a single numerical score — the loss — that quantifies the model's error on a given batch of training data. The entire training process is aimed at minimizing this loss value over time.
How It Works
After the network produces a prediction, the loss function compares it to the ground truth label and computes an error score. Mean Squared Error (MSE) is commonly used for regression problems — it calculates the average squared difference between predicted and actual values. Cross-Entropy Loss is used for classification — it penalizes confident wrong predictions more severely than uncertain ones. The loss value is then used by the optimizer via backpropagation to update the network's weights.
Common Functions
Mean Squared Error Cross-Entropy Loss Binary Cross-Entropy Huber Loss
🔧
Component 06
Optimization Algorithms
Optimization algorithms are the methods used to update the network's weights during training in order to minimize the loss function. They define how the weight adjustments are calculated and applied after each training step. The choice of optimizer significantly affects training speed, stability, and the final performance of the model.
How It Works
Gradient Descent is the foundational optimization approach — it calculates the gradient (direction of steepest increase) of the loss function with respect to each weight, and then moves each weight in the opposite direction to decrease the loss. The learning rate controls how large each step is. Adam (Adaptive Moment Estimation) improves on basic gradient descent by maintaining adaptive learning rates for each parameter and using momentum to accelerate training. Adam is the most widely used optimizer in modern deep learning.
Common Optimizers
Gradient Descent SGD Adam RMSprop AdaGrad
Neural Network Playground
Hands-On Exploration

Experimenting with the Neural Network Playground

Using the TensorFlow Neural Network Playground (playground.tensorflow.org), I experimented with different data types, noise levels, features, and neuron layer configurations to observe how they affect training and model performance.

The playground provided an immediate, visual way to connect the theoretical components of neural networks to their practical effects — transforming abstract concepts into observable behaviors.

🗂️

More layers enabled the network to classify non-linear data (spiral patterns) that a single layer could not separate — demonstrating why deep networks are needed for complex tasks.

⚡

More neurons per layer increased model capacity and reduced training error, but too many neurons with too little data led to overfitting — the model memorized training data instead of generalizing.

📈

ReLU activation consistently trained faster and reached lower loss values compared to Sigmoid — especially in deeper networks, where Sigmoid suffered from vanishing gradient problems.

🔧

Higher learning rates initially converged faster but caused instability and oscillation around the minimum, while smaller learning rates produced smoother, more reliable convergence.

Summary Slide
Key Insights & Takeaways

Why Visualizing Neural Networks Matters

Neural networks are often treated as black boxes — visualizing their components transforms them from opaque systems into understandable, interpretable tools.

Structure Enables Learning

The layered architecture — input, hidden, output — allows neural networks to build increasingly abstract representations of data, enabling them to solve problems no single layer could handle.

Weights Are Knowledge

All the knowledge a neural network acquires from training data is encoded in its weights. Training is the process of discovering the right set of weights through iterative error correction.

Components Are Interdependent

No single component works in isolation. The activation function, loss function, and optimizer must all be chosen to work together — poor choices in any one component degrade the entire system.

Key Insight from This Exercise

Creating this visual framework revealed that neural networks are not magical black boxes — they are precise, modular systems built from well-defined mathematical components. Each layer performs a specific transformation, each neuron computes a specific function, each weight encodes a specific learned relationship, and each optimizer follows a specific mathematical strategy. Understanding these components individually is the first step toward building, debugging, and improving neural network models in real-world AI applications. This understanding directly connects to why deep learning has become the dominant paradigm across computer vision, NLP, and generative AI.

Artifact 2 — Portfolio Reflection
Learning Experience & Neural Network Insights

Reflection on the Neural Network Components Artifact

The Process I Used

Building this artifact required a different kind of thinking than the ML Algorithms Framework. Rather than classifying algorithms across a broad landscape, I had to go deep into the internal mechanics of a single system. I began by reviewing course resources on neural network architecture, then moved to hands-on experimentation in the Neural Network Playground to observe how changes to layers, neurons, activation functions, and learning rates affected model behavior in real time. I then translated those observations into detailed written explanations for each of the six components, designed the SVG architecture diagram from scratch, and synthesized my findings into the summary slide.

Key Decisions Made

The most significant decision was how to explain each component — not just as an isolated definition, but in terms of its role within the system and how it interacts with the other components. For example, simply defining the loss function as "a measure of error" would be accurate but incomplete without explaining how it connects to the optimizer through backpropagation. I chose to write each explanation in two parts: what it is, and how it actually works within the training loop. This approach produced much more useful explanations than definitions alone.

Challenges I Overcame

The most challenging aspect was explaining weights and optimization together without the explanation becoming circular. Weights are updated by the optimizer, the optimizer uses the loss function, the loss function depends on the network's predictions, and predictions depend on the weights. Explaining this cycle in a linear, step-by-step written format required careful sequencing. I resolved this by explaining the forward pass (input → prediction) and backward pass (loss → weight update) as two clearly distinct phases of the training loop, which made the circular dependency understandable rather than confusing.

What I Learned

The Neural Network Playground was unexpectedly educational. Seeing the decision boundary update in real time as I changed the number of neurons or switched from Sigmoid to ReLU made abstract mathematical concepts immediately tangible. I observed firsthand that more layers do not always mean better performance — with insufficient data or too high a learning rate, additional complexity led to unstable training or overfitting. This directly reinforced the course concept that model architecture choices and hyperparameter tuning are as important as choosing the right algorithm type.

How My Skills Developed

This artifact developed a qualitatively different skill set than Artifact 1. Where the first artifact built my ability to navigate the broad landscape of ML algorithms, this one built my ability to go inside a single model and understand its mechanics at a component level. Together, these two artifacts represent complementary competencies — breadth and depth — that are both essential for professional AI/ML practice. Moving forward, I intend to apply this component-level understanding when implementing and debugging neural network models in future assignments, using the knowledge of how weights, activation functions, and optimizers interact to make more informed decisions about architecture choices and troubleshooting strategies.
