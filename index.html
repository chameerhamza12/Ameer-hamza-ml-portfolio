Title

Machine Learning Algorithm Visual Framework: A Comprehensive Classification of 11 Key AI/ML Algorithms by Learning Type, Application Domain, and Real-World Use Cases

Introduction

This artifact is a professional portfolio piece created for AIML-501: Model Development at Indiana Wesleyan University. It presents a visually structured framework designed to help practitioners, students, and employers quickly understand the landscape of machine learning algorithms — how they are categorized, where they are applied, and how they relate to each other across the rapidly evolving AI field.

Description

The framework covers 11 key machine learning algorithms spanning supervised and unsupervised learning categories, organized across four major application domains: Tabular Data, Computer Vision, Natural Language Processing, and Generative AI. Each algorithm entry includes its learning type, applicable domains, a plain-English explanation of how it works, and real-world use cases. The framework also includes a relationship map showing how algorithms connect across domains, a detailed classification rationale explaining the decision-making process behind each categorization, and a comprehensive reflection on the learning experience.

Objective

To develop a clear, accurate, and visually compelling reference framework that demonstrates mastery of key ML algorithm types and their practical applications — fulfilling the course learning objectives of identifying, classifying, and communicating algorithmic concepts across AI domains. This artifact also serves as a professional portfolio piece to showcase AI/ML knowledge to potential employers and collaborators.

Process

The framework was developed through a structured multi-step process: (1) reviewing course materials and background resources on supervised learning, unsupervised learning, computer vision, NLP, and generative AI; (2) selecting 11 representative algorithms that cover the breadth of the field; (3) researching each algorithm's mechanism, strengths, and real-world applications; (4) designing a classification system based on learning type and domain; (5) building the visual framework using HTML and CSS; and (6) writing classification rationale and reflection to demonstrate critical thinking about the decisions made.

Tools and Technologies Used

HTML5 and CSS3 for the visual portfolio framework design; JavaScript for interactive animations and smooth scroll behavior; Google Fonts (Instrument Serif + Bricolage Grotesque) for professional typography; GitHub Pages for free, public portfolio hosting and shareable link generation; course readings and AI/ML reference materials for content accuracy.

Value Proposition

The combination of visual design, technical accuracy, and written rationale makes it a genuinely useful reference that employers and peers can learn from, not just a static document.

Relevance

As AI and ML continue to transform every industry, the ability to navigate and communicate the algorithm landscape is a core professional competency. This artifact demonstrates that ability directly. It is relevant not only to this course but to any future role involving ML model selection, AI strategy, data science consulting, or technical communication — making it a durable and versatile portfolio asset that grows in value as the field evolves.

Explanation of Classification Process
Why I Classified Algorithms This Way

Classification Rationale & Critical Thinking

Building this framework required deliberate classification decisions at three levels: (1) learning type — supervised vs. unsupervised, (2) application domain — tabular, computer vision, NLP, and/or generative AI, and (3) algorithm selection — which 11 algorithms best represent the breadth of the field. Below is the explicit rationale for each major decision, demonstrating the critical thinking behind the framework's structure and organization.

Supervised Linear Regression

Classified as supervised because it requires a labeled continuous target variable to train. Assigned to tabular data because its inputs are structured numerical features. Included specifically because the assignment explicitly names it as a foundational supervised learning example — and because it underpins the mathematics of neural networks, making it the logical starting point of the framework.

Supervised Decision Tree & Random Forest

Both supervised tabular algorithms. Random Forest was included as a distinct entry rather than a variation of Decision Tree because it fundamentally changes the bias-variance tradeoff through ensemble aggregation — overcoming the core weakness (overfitting) of a single tree. This distinction illustrates an important architectural evolution that is worth highlighting explicitly in any comprehensive ML framework.

Supervised Gradient Boosting

Supervised tabular only — not CV or NLP — because Gradient Boosting is specifically optimized for structured, feature-engineered datasets. Unlike neural networks, it does not naturally process raw pixels or token sequences. Its placement reflects where it genuinely excels in practice rather than where it could theoretically be applied with extensive preprocessing.

Supervised Support Vector Machine

Placed across both tabular and computer vision because SVMs were the dominant image classifier before CNNs — particularly for handwriting recognition — and remain applicable in low-data CV scenarios. This cross-domain placement reflects meaningful historical accuracy and practical relevance rather than assigning it to only its most common current use case.

Supervised CNN

Primary domain is computer vision — CNNs were specifically designed to exploit spatial structure in images. Also placed under generative AI because CNNs form the architectural backbone of GAN generators and early diffusion models. This cross-domain classification is accurate and educationally important: it shows how the same architecture can serve both discriminative and generative purposes.

Supervised Transformer (BERT/GPT)

Placed in both NLP and generative AI because BERT and GPT represent the same Transformer architecture applied to two distinct objectives — understanding vs. generation. Classifying them together under one entry accurately reflects how Transformers have become the unifying architecture across NLP and generative AI, rather than two separate algorithm families.

Supervised GAN & Diffusion Model

Both classified as supervised — not unsupervised — because they require labeled training signals. GANs use real/fake binary labels; Diffusion Models supervise the denoising process at each noise level. This corrects a common misconception that generative models are unsupervised. Both are placed in generative AI as primary domain and computer vision as secondary, reflecting their most prominent real-world deployments.

Unsupervised K-Means & PCA

Both unsupervised — neither requires class labels during training. K-Means discovers natural groupings without predefined categories. PCA finds axes of maximum variance without knowing what the data represents. PCA was placed across tabular, computer vision, and NLP because dimensionality reduction is universally applicable — it reduces noise in tabular features, compresses image pixel spaces, and condenses word embedding dimensions.

Learning Experience & Growth

Reflection on Building This Framework

The Process I Used to Complete This Artifact

Creating this framework required a deliberate multi-stage process. I began by reviewing the course materials and the background resources provided — including readings on supervised learning, unsupervised learning, computer vision, NLP, and generative AI. I then identified 11 algorithms that collectively represent the full breadth of the ML landscape, ensuring coverage across all four required domains. For each algorithm, I researched not only how it works mechanically but also where it is actually deployed in industry, since real-world relevance is what makes this framework genuinely useful. I then designed the classification structure — deciding first on the two primary categories (learning type and application domain) — and built the visual framework using HTML and CSS to ensure it would be accessible as a professional portfolio artifact via a public link. Finally, I wrote the classification rationale and reflection to document the thinking behind each decision, which deepened my own understanding significantly.

Key Decisions I Made Along the Way

Several important decisions shaped the final framework. First, I chose to classify GANs and Diffusion Models as supervised rather than unsupervised — a decision that challenged the common assumption that generative models are unsupervised. Writing the rationale for this forced me to think carefully about what "supervised" actually means at the training level. Second, I decided to include Linear Regression as Algorithm #1 even though it is the simplest model in the framework, because it anchors the entire structure and connects to neural networks mathematically. Third, I chose to place PCA across three domains (tabular, CV, and NLP) rather than limiting it to one, because restricting it would have misrepresented its actual versatility and cross-domain utility in practice.

Challenges I Overcame

The most challenging aspect was accurately placing algorithms across multiple domains without oversimplifying or misrepresenting them. SVMs, for instance, are primarily associated with tabular classification today, yet excluding them from computer vision would have erased an important historical and technical truth. Similarly, deciding how to handle algorithms like PCA — which is genuinely applicable everywhere — required developing a principled approach to cross-domain classification rather than defaulting to a single label for the sake of simplicity. Another challenge was understanding the distinction between GANs and Diffusion Models deeply enough to explain why Diffusion Models now produce higher-quality outputs despite both belonging to the generative AI category. Researching this distinction pushed me beyond surface-level understanding into the actual mechanics of each approach.

What I Learned About the Subject

This assignment solidified my understanding of the supervised vs. unsupervised distinction — a difference that is not just categorical but determines the entire training pipeline, evaluation methodology, and appropriate use case. I also gained a much clearer grasp of why Generative AI represents a genuine paradigm shift: traditional ML algorithms optimize for prediction accuracy on existing data distributions, whereas generative models learn to sample from and extend those distributions to produce novel outputs. Most importantly, I discovered how deeply interconnected the algorithm ecosystem is — Generative AI is not a separate field but is built directly on the same supervised architectures (CNNs, Transformers) that power classical computer vision and NLP. Mastering the fundamentals is not just preparatory; it is directly transferable to the most advanced AI systems in deployment today.

How My Skills Developed and Future Applications

Before this assignment, I could name these algorithms but could not confidently explain the reasoning behind choosing one over another for a given problem. Building this framework changed that. I can now approach any ML problem and systematically ask: Is the data labeled? What domain is it from? What output is needed? And this framework maps directly to those answers. This is one of the most practically valuable skills in the AI/ML profession — model selection — and I now have a solid foundation for it. Moving forward, I intend to use this framework as a living reference throughout the course and my career. As I gain hands-on experience with each algorithm through future assignments, I will return to this artifact and deepen my understanding of each entry. I also plan to continue developing my portfolio with additional artifacts that demonstrate applied competence, not just theoretical knowledge — because in a competitive AI/ML landscape, showing your work is what sets you apart.
