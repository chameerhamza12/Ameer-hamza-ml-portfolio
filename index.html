10 Key Machine Learning Algorithms
01 Supervised
Decision Tree
Classifies data by learning a hierarchy of if-then rules derived from input features. Splits the dataset at each node using Gini impurity or information gain until pure leaf nodes are reached. Highly interpretable and requires no feature scaling.
Tabular Data
Real-World Use Cases
Credit risk scoring Medical diagnosis Customer churn prediction
02 Supervised
Random Forest
An ensemble method that builds hundreds of Decision Trees on random data subsets and aggregates their predictions by majority vote. Reduces overfitting dramatically while improving accuracy and handling high-dimensional data robustly.
Tabular Data
Real-World Use Cases
Fraud detection Disease prediction Stock market forecasting
03 Supervised
Gradient Boosting (XGBoost)
Builds an ensemble of weak learners sequentially, where each new tree corrects the residual errors of the previous one using gradient descent. Achieves state-of-the-art performance on structured tabular data and dominates competitive ML benchmarks.
Tabular Data
Real-World Use Cases
Kaggle competitions Sales forecasting Insurance pricing
04 Supervised
Support Vector Machine
Finds the optimal hyperplane that maximizes the margin between classes in feature space. Uses kernel functions (RBF, polynomial) to handle non-linearly separable data. Effective in high-dimensional spaces and robust to outliers.
Tabular Data Computer Vision
Real-World Use Cases
Email spam detection Handwriting recognition Bioinformatics
05 Supervised
Convolutional Neural Network
Uses stacked convolutional layers to automatically learn spatial feature hierarchies from raw pixel data — edges, textures, shapes, and semantic objects. Pooling layers reduce spatial dimensions while preserving key features. Powers virtually all modern computer vision systems.
Computer Vision Generative AI
Real-World Use Cases
Image classification Object detection (YOLO) Medical imaging Face recognition
06 Supervised
Transformer (BERT / GPT)
Processes entire input sequences in parallel using a self-attention mechanism that weighs the contextual relevance of every token relative to every other. BERT uses bidirectional encoding for understanding tasks; GPT uses autoregressive decoding for generation. Foundation of all modern LLMs.
NLP Generative AI
Real-World Use Cases
ChatGPT / Claude Sentiment analysis Machine translation Code generation
07 Supervised
Generative Adversarial Network
Pits two neural networks — a Generator and a Discriminator — against each other in a zero-sum game. The Generator creates synthetic samples; the Discriminator learns to distinguish real from fake. Through adversarial training, the Generator progressively produces increasingly realistic outputs.
Computer Vision Generative AI
Real-World Use Cases
Deepfake generation Image super-resolution Synthetic data creation
08 Supervised
Diffusion Model
Learns to generate data by reversing a gradual noising process. During training, Gaussian noise is progressively added to images until they become pure noise. The model learns to denoise step by step, guided by text or other conditioning signals. Produces higher-quality, more diverse outputs than GANs.
Computer Vision Generative AI
Real-World Use Cases
Stable Diffusion DALL·E image generation Drug molecule design
09 Unsupervised
K-Means Clustering
Partitions unlabeled data into k clusters by iteratively assigning each point to its nearest centroid and recalculating centroids until convergence. The number of clusters k must be defined in advance. Effective for discovering natural groupings in exploratory analysis.
Tabular Data
Real-World Use Cases
Customer segmentation Document clustering Anomaly detection
10 Unsupervised
Principal Component Analysis
Reduces high-dimensional data to fewer dimensions by identifying principal components — orthogonal axes of maximum variance. Preserves the most important structural information while discarding noise. Widely used as a preprocessing step before applying other ML algorithms.
Tabular Data Computer Vision NLP
Real-World Use Cases
Face recognition preprocessing Gene expression analysis Data visualization
11 Supervised
Linear Regression
Models the relationship between input features and a continuous numerical output by fitting the best-fit line (or hyperplane) that minimizes the sum of squared errors. One of the oldest and most interpretable ML algorithms, forming the mathematical foundation for more complex neural network architectures.

Rubric: Explanation of Classification Process
Why I Classified Algorithms This Way

Classification Process & Rationale

Building this framework required making deliberate classification decisions at three levels: (1) learning type — supervised vs. unsupervised, (2) application domain — tabular, computer vision, NLP, and/or generative AI, and (3) algorithm selection — which 11 algorithms best represent the breadth of the field. Below is the explicit rationale for each major decision, demonstrating the critical thinking behind the framework's structure.

Supervised Linear Regression

Classified as supervised because it requires labeled training data — specifically, a continuous numerical target variable (e.g., house prices). It belongs to tabular data because its inputs are structured numerical or categorical features in rows and columns. It was included because the assignment explicitly names it as a foundational supervised learning example, and it underpins the mathematics of more complex models including neural networks.

Supervised Decision Tree & Random Forest

Both classified as supervised tabular algorithms because they learn from labeled datasets to produce classification or regression outputs on structured data. Random Forest was included as a distinct entry because it fundamentally changes the bias-variance tradeoff through ensemble aggregation — it is not simply a larger Decision Tree, but a qualitatively different approach that overcomes the core weakness (overfitting) of a single tree.

Supervised Gradient Boosting (XGBoost)

Classified as supervised tabular because it learns from labeled data through sequential error correction. Placed in tabular data (not CV or NLP) because Gradient Boosting is specifically optimized for structured, feature-engineered datasets and does not naturally handle raw image pixels or token sequences. It achieves state-of-the-art results on tabular benchmarks, making it the go-to algorithm for real-world structured data problems.

Supervised Support Vector Machine

Classified as supervised because it requires labeled class boundaries to learn a separating hyperplane. Placed across both tabular data and computer vision because SVMs were the dominant image classifier before CNNs, particularly for handwriting recognition (MNIST), and remain applicable in low-data computer vision scenarios. This cross-domain placement reflects meaningful historical and practical relevance.

Supervised CNN

Classified as supervised because training requires labeled image datasets. Placed in computer vision as its primary domain — CNNs were designed specifically to exploit spatial structure in images through learned convolutional filters. It also appears under generative AI because CNNs form the architectural backbone of GAN generators and early image diffusion models, making the cross-domain classification accurate and educationally meaningful.

Supervised Transformer (BERT / GPT)

Classified as supervised because Transformers are trained on labeled or self-supervised objectives (next-token prediction, masked language modeling). Placed in both NLP and generative AI because BERT targets language understanding tasks while GPT targets language generation — they represent the same architecture applied to two distinct objectives. This dual-domain classification reflects how Transformers have become the unifying architecture across NLP and generative AI.

Supervised GAN & Diffusion Model

Both classified as supervised (not unsupervised) because they require labeled training data — GANs are trained with real/fake binary labels, and Diffusion Models are trained by supervising the denoising process at each noise level. Both are placed in generative AI as their primary domain and computer vision as a secondary domain because their most prominent applications involve generating photorealistic images (Stable Diffusion, DALL·E, StyleGAN).

Unsupervised K-Means & PCA

Both classified as unsupervised because neither requires class labels during training. K-Means discovers natural groupings without any predefined categories. PCA finds axes of maximum variance without knowing what the data represents. PCA was placed across tabular, computer vision, and NLP because dimensionality reduction is a universally applicable preprocessing step — it reduces noise in tabular features, compresses image pixel spaces, and condenses word embedding dimensions before downstream modeling.

Portfolio Reflection
Learning Experience & Insights

Reflection on Building This Framework

What I Found Most Interesting

The most intellectually engaging aspect of this assignment was uncovering how deeply interconnected these algorithms are — they are not isolated tools but a coherent ecosystem built on shared mathematical foundations. What surprised me most was how Generative AI, which appears to be a recent and entirely distinct field, actually builds directly on foundational supervised learning architectures like CNNs and Transformers. A Diffusion Model, for instance, is essentially a supervised denoising task applied iteratively, and a GAN is a pair of supervised networks engaged in adversarial training. Recognizing these structural continuities changed how I understand the field: mastering the fundamentals is not just preparatory — it is directly transferable to the most advanced AI systems being deployed today.

Challenges Encountered

The most challenging aspect was accurately placing algorithms within multiple application domains without oversimplifying. For example, SVMs are primarily associated with tabular classification, yet they have historically been used in computer vision for handwriting recognition before CNNs became dominant. Similarly, PCA is broadly applicable across tabular data, computer vision preprocessing, and even NLP (for dimensionality reduction in word embedding spaces), which required careful thought about how to represent cross-domain relevance without making the framework cluttered. Choosing the right level of detail — enough to be informative but not so much as to obscure the relationships — required several iterations of the framework design.

Knowledge Gained

This assignment solidified my understanding of the distinction between supervised and unsupervised learning — a distinction that is not just categorical but determines the entire training pipeline, evaluation methodology, and appropriate use case. I also gained a clearer grasp of why Generative AI represents a paradigm shift: traditional ML algorithms optimize for prediction accuracy on existing data distributions, whereas generative models learn to sample from and extend those distributions to produce novel outputs. Understanding this difference is critical for selecting the right approach in applied ML projects.

Connection to Course Concepts & Future Applications

This framework serves as a practical decision-making guide for model selection — one of the most important applied skills in the AI/ML profession. When approaching a new problem, the first questions a practitioner must answer are: Is my data labeled? What domain is it from? What is the desired output? This visual framework directly maps those questions to algorithm choices. Moving forward in the course, I intend to use this reference when evaluating which algorithms are appropriate for the datasets and problems presented in subsequent assignments. As AI adoption accelerates across industries, the ability to navigate this landscape of algorithms confidently and communicate their distinctions clearly is a core professional competency I am actively developing.
